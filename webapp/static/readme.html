推荐流程

  1. 先进入 Web Studio，加载一首示例歌曲，熟悉时间线和右侧编辑区。
  2. 上传自己的 mp3、wav 或 flac，填写歌曲名后开始分析。
  3. 检查自动识别的音乐段落，必要时在右侧结构编辑器里修正。
  4. 检查动作时间线，替换不合适的应援动作或调整起止时间。
  5. 可选：加载本地 MV / 现场视频，让右上角视频和歌曲用同一个播放条同步播放。
  6. 添加备注，保存编辑，再导出 JSON、Markdown 或教学视频。
  7. 需要新 MIX / Call 时，进入 MIX Builder 创建并保存到本地动作库。

查看示例歌曲

  1. 打开 Web Studio。
  2. 在右上角的“示例歌曲”下拉框选择一首歌。
  3. 点击“加载”。
  4. 页面会加载示例分析结果，包含音乐段落、动作时间线、canvas 预览和音频播放器。

示例不会改动你的本地用户数据，适合用来快速确认页面和播放器是否正常。

公开仓库默认不包含商业歌曲 MP3。若你有合法来源的示例音频，请放到 example_audio/，文件名使用歌曲 ID，例如 athiscode.mp3、soundscape.mp3。重启 YesTiger 后，示例播放器会自动使用对应本地音频。

分析自己的歌曲

  1. 把音频文件拖到左侧“拖放音频”区域，或者点击该区域选择文件。
  2. 在“歌曲名”输入框填写标题；不填时会使用文件名。
  3. 点击“开始分析”。
  4. 分析完成后，左侧会显示时长、速度、小节数和动作数，中间会显示 canvas、播放器和动作时间线。

第一次分析可能比较慢，因为模型缓存、音频结构分析和特征提取都需要准备。等待时不要关闭终端窗口。

修改音乐段落

  1. 歌曲加载后，右侧会出现“结构编辑器”。
  2. 每一行代表一个音乐段落，可以修改开始时间、结束时间和段落标签。
  3. 常用标签包括 intro、verse、pre_chorus、chorus、bridge、solo、instrumental_break、outro。
  4. 需要拆分或补充段落时，点击“添加段落”。
  5. 调整完成后点击“保存编辑”。

音乐段落会影响导出的结果，也会影响你后续理解动作为什么安排在某个位置。

修改应援动作和种类

  1. 中间的“动作时间线”展示当前歌曲的应援安排。
  2. 可以直接改每行动作的开始时间、结束时间、段落、动作、风险和文本。
  3. 动作输入框支持搜索；输入关键词后，从下拉结果里选择一个动作。
  4. 动作种类包括 keepspace、rhythmcall、mix 和 underground_gei。
  5. 使用时间线上方的筛选按钮，可以只看某一种动作。
  6. 使用每行的加号和删除按钮，可以插入或删除动作行。

改完动作后，记得点击右侧的“保存编辑”，否则刷新页面后当前修改不会写入本地结果。

自定义备注

  1. 歌曲加载后，右侧会出现“备注 Notes”。
  2. 点击“添加备注”，填写备注的开始时间、结束时间和文字内容。
  3. 备注适合记录站位、颜色、特殊手势、现场提醒或个人练习要点。
  4. 完成后点击“保存备注”。

备注会跟随当前歌曲结果保存在本地，可以和 JSON 结果一起备份。

加载本地视频

  1. 歌曲加载后，在 canvas 下方点击“选择歌曲视频”。
  2. 选择本机上的 MV、现场视频或练习参考视频。
  3. 视频会显示在教学画面右上角，并跟随歌曲音频一起播放、暂停、拖动和变速。
  4. 不需要视频时，点击“移除视频”。

视频文件只在本地浏览器和导出接口中使用，不会写入 GitHub。导出 MP4 时，当前加载的视频会一起合成进右上角画面。

自定义 MIX / Call

  1. 从首页进入 MIX Builder。
  2. 填写 MIX 名称、类别、风险、小节数和中文别名。
  3. 在“小节内容”里逐小节填写喊词。
  4. 选择适用场景，例如 chorus、intro、bridge 或 instrumental_break。
  5. 右侧会实时预览生成的动作定义。
  6. 点击“保存到动作库”，这个动作会进入本机用户动作库。

保存后的自定义动作会写入 webapp_runs/custom_actions/。回到 Web Studio 后，它会出现在动作搜索里，可以被手动选入时间线。

动作库参考

  1. 从首页进入 MIX & Call Library。
  2. 按名称、别名、喊词或 ID 搜索动作。
  3. 用类别和风险筛选 MIX、节奏 Call、地下艺或留白。
  4. 查看中日文名称、内容、小节长度、适用场景和来源。

该页面直接读取当前 knowledge/call_mix_library.json 和本地自定义动作，适合编辑时间线前快速查阅。

保存与导出

  • “保存编辑”会保存音乐段落、动作时间线和备注到当前歌曲结果。
  • “保存备注”会单独保存备注列表，备注会显示在教学画面左上角，并参与 MP4 导出。
  • “导出 JSON”会下载完整结构化结果，适合继续处理或备份。
  • “导出 MD”会下载 Markdown 版 callbook，适合阅读和分享文字方案。
  • “下载教学视频”会生成 MP4，包含当前教学画面、右上角本地视频、原曲音频、call / MIX 声音和备注。

数据位置

上传音频
webapp_runs/uploads/
分析结果和保存后的编辑
webapp_runs/jobs/
用户自定义 MIX / Call
webapp_runs/custom_actions/
可选示例音频
example_audio/
模型与特征缓存
.hf/ 和 webapp_runs/feature_cache/

换电脑或重装前，备份 webapp_runs/ 就能保留用户上传、编辑结果、备注和自定义动作。
